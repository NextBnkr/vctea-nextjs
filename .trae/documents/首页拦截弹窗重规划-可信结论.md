# 首页拦截弹窗重规划（可信结论提示）

## Summary
- 目标：把当前“底部提示后 900ms 自动跳转 intake”的轻提示流程，改为“拦截弹窗”流程，避免用户还没读完就被动跳页。
- 方案：在首页 `app/page.tsx` 内新增强拦截弹窗状态与渲染，未满足核心资料时点击“开始查询”先弹窗说明，再由用户主动确认跳转。
- 用户偏好：采用强拦截；文案使用委婉礼貌语气，明确“VC 资料按商业机密处理，会结合真实情况脱敏加工后给出结论，因此需要先完善资料”。

## Current State Analysis
- 当前跳转逻辑集中在 [page.tsx](file:///Users/maha/vctea-nextjs/app/page.tsx#L184-L207) 的 `goSearch`：
  - 已填核心资料：直接写入 `vccha_run_payload` 并跳 `'/result'`。
  - 未填核心资料：`setShowGuide(true)` + 写入 `vccha_pending_org_query` + `setTimeout(900)` 后自动跳 `'/intake?org=...'`。
- 当前提示 UI 为页面内蓝底说明块 [page.tsx](file:///Users/maha/vctea-nextjs/app/page.tsx#L276-L281)，不具备拦截能力。
- 仓库没有现成 `Dialog/Modal` 组件；`components/ui` 只有 `button/input/card/...` 基础组件，因此本次在首页内按现有 Tailwind 风格实现轻量弹窗最稳妥。

## Proposed Changes

### 1) 改造首页状态与流程（核心）
- 文件：`app/page.tsx`
- 变更点：
  - 移除/替换 `showGuide` 相关逻辑，不再使用“底部提示 + 900ms 自动跳转”。
  - 新增拦截弹窗可见状态（例如 `showInterceptModal`）与“待处理机构名”状态（或直接复用 `trimmedQuery`）。
  - 调整 `goSearch` 分支：
    - 核心资料完整：保持原行为（直达 result）。
    - 核心资料不完整：写入 `vccha_pending_org_query`，打开拦截弹窗，不立即路由跳转。
  - 新增“确认去完善资料”动作：由弹窗主按钮触发 `router.push('/intake?org=...')`。
- 原因：
  - 解决“文案未读完就跳页”的体验问题。
  - 让用户对跳转有明确知情与主动确认。

### 2) 新增拦截弹窗 UI（强拦截）
- 文件：`app/page.tsx`
- 变更点：
  - 在页面根层添加固定定位弹窗层（遮罩 + 内容卡片），视觉风格延续 amber/slate、轻玻璃与弱边框。
  - 弹窗内容采用礼貌说明文案，明确三点：
    1. 资料按商业机密处理；
    2. 会结合真实情况进行脱敏加工；
    3. 为保证结论可信需先完善资料。
  - 仅提供一个主动作按钮（强拦截）：`我知道了，去完善资料`（最终文案可在实现时微调）。
  - 不提供关闭按钮与次按钮；点击遮罩不关闭，保证“强拦截”一致性。
- 原因：
  - 与“强拦截跳转”决策一致，减少状态分叉和误触。

### 3) 状态与交互细节收口
- 文件：`app/page.tsx`
- 变更点：
  - 修正 `redirecting` 的状态时机：当进入“弹窗等待确认”时，不让“开始查询”长期处于 `处理中...` 卡死状态。
  - 防重复触发：弹窗打开期间禁用重复查询触发（可通过按钮 `disabled` 或逻辑短路）。
  - 保留现有 Enter 提交行为，但在未完善资料时统一进入弹窗分支。
- 原因：
  - 避免因为流程改造引入新的交互 bug。

## Assumptions & Decisions
- 决策：本次只改首页 `app/page.tsx`，不引入新的通用 Dialog 组件，控制改动范围与回归风险。
- 决策：强拦截语义下，不提供“暂不填写/关闭弹窗”出口。
- 假设：现有 `intake` 页继续通过 `org` query + `vccha_pending_org_query` 兜底接收机构名，保持兼容。
- 决策：核心资料判定规则（`oneLineSummary + fundingAmount + teamBackground`）保持不变，不调整业务口径。

## Verification
- 手工验证（桌面 + 移动视口）：
  - 未填核心资料：输入机构后点“开始查询”/按 Enter，出现拦截弹窗，不会自动跳页。
  - 弹窗文案完整可读，且点击遮罩/空白区不会关闭。
  - 点击主按钮后跳转到 `/intake?org=xxx`，并在 intake 的目标机构中自动带入该机构。
  - 已填核心资料：仍直接进入 `/result`，不出现弹窗。
  - 按钮文案与 loading 状态正确，不出现“处理中...”卡住。
- 回归检查：
  - 输入建议下拉、`先完善资料` 次按钮、placeholder 打字效果不受影响。

# 无边框 UX 重构计划（首页 + 结果页）

## 1) Summary
- 目标：将首页与结果页从“重卡片/重容器”的拥挤观感，重构为“轻玻璃感、无边框、明显减密”的移动优先体验。
- 约束：主题颜色保持现有暖橙系与深蓝文字体系，不改业务逻辑与数据结构。
- 范围：仅重构页面布局与交互呈现层，覆盖 `/app/page.tsx` 与 `/app/result/page.tsx`；允许中度交互调整（模块重排、信息折叠）。

## 2) Current State Analysis

### 首页（`/app/page.tsx`）
- 当前结构是大容器 + 多层卡片嵌套（外层 `rounded-[30px]` + 内层 `Card` + 提示色块），视觉层级过多，导致拥挤感明显。
- 表单块使用连续“标题 + 输入框 + 必填 Badge + 提示”，同屏信息密度高；步骤导航仅展示 `1/4`，上下文弱。
- 底部操作条是粘性白底容器，视觉上再次形成“盒中盒”。
- 当前样式以 Tailwind 原子类为主，易于局部替换；无独立模块样式耦合。

### 结果页（`/app/result/page.tsx`）
- 头部、进度、结果列表均在大白卡中，且每条机构结果再嵌套次级卡片，层级过深。
- 每条结果默认一次性展开“亮点/关注/补充”全文，移动端滚动负担大。
- 操作按钮（返回/重评估）与状态信息同层竞争注意力，主次关系不够清晰。

### 共享基础组件（`/components/ui/*.tsx`）
- `Input`、`Textarea`、`Card` 已是“透明边框 + 阴影”的轻量实现，说明无边框方案可优先在页面布局层落地，避免改动全局组件造成连带影响。

## 3) Proposed Changes

### A. 首页重构（`/app/page.tsx`）
- 采用“单列内容流 + 分段玻璃面板”替代“多层卡片嵌套”：
  - 保留背景暖色渐变，但弱化外层大白盒，主体改为更轻的半透明段落区块。
  - Hero 区与步骤区分离：标题区更短、更聚焦；步骤信息改为“水平轻量进度条 + 当前步骤文案”。
- 显著减密输入流程：
  - 每步只保留关键说明与字段，辅助文案降噪并改为可折叠/弱化提示。
  - 必填标识收敛为统一样式（减少视觉噪点），字段间距扩大，提升扫读节奏。
- 历史记录区重构为“抽屉感列表”：
  - 默认折叠，展开后使用低对比行项，减少与主表单竞争。
- 底部操作区无边框化：
  - 去掉强白底与厚阴影，改为轻玻璃悬浮条；主按钮保持主题橙色，次按钮灰阶弱化。

### B. 结果页重构（`/app/result/page.tsx`）
- 头部信息重排为“标题区 + 行动区 + 进度条”三段流式布局：
  - 保留“返回修改表单/重新评估”，但降低次操作视觉权重，突出主要流程状态。
- 结果列表改为“无边框信息块 + 默认折叠详情”：
  - 每个机构先展示：机构名、优先级、结论标签、置信度。
  - “匹配亮点/需关注事项/建议补充信息”进入可展开区域（accordion 样式），默认收起或仅展开首条。
  - 长文本改段落分隔与更高行高，避免密集文本墙。
- 空结果与加载态统一为轻量信息面板，减少大面积实底容器。

### C. 视觉与交互一致性（同改上述两页）
- 保持原主题色：
  - 主强调继续使用 amber 系；文本保持 slate 深浅层级。
  - 结论标签沿用现有语义配色策略（emerald/amber/sky/slate/rose）。
- 建立统一无边框语言：
  - 减少“实底 + 重阴影 + 大圆角”叠加次数。
  - 通过留白、字号、字重、透明度和微弱阴影建立层次。

## 4) Assumptions & Decisions
- 已确认：覆盖首页与结果页，采用“轻玻璃感”，信息密度“明显减密”，允许中度交互调整。
- 决策：不改接口调用、数据解析逻辑、字段定义与存储键（`localStorage/sessionStorage`）。
- 决策：优先在页面层改造，不先改 `components/ui` 基础组件，降低回归风险。
- 决策：移动端优先（基于当前截图场景），同时保留现有 `md` 断点行为并做自适配优化。

## 5) Verification Steps
- 视觉验证：
  - 启动本地页面后检查首页与结果页在移动宽度与桌面宽度下的层级、留白、可读性是否达成“无边框 + 减密”目标。
  - 对比改造前后：同屏信息块数量减少、主次动作更清晰、长文本阅读负担降低。
- 交互验证：
  - 首页完整走通 `1/4 -> 4/4 -> 查看匹配结果`，校验必填校验、历史记录展开/回填、上下步切换正常。
  - 结果页校验“返回修改表单”“重新评估”可用；折叠展开不影响原有结果渲染。
- 稳定性验证：
  - 确认 TypeScript/构建无新增报错，页面无明显布局溢出与交互阻断。

## 6) Implementation Order
1. 重构首页整体布局骨架与步骤头部信息层级（先解决拥挤根因）。
2. 重构首页表单区与底部操作区（完成无边框交互主流程）。
3. 重构结果页头部、进度区与列表容器（统一视觉语言）。
4. 增加结果项折叠交互与文本排版优化（完成显著减密）。
5. 全链路回归验证与响应式检查。

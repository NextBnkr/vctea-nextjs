# DemoApp 文案重构计划（MVP / 用户中心 / AI FA 价值叙事）

## Summary

* 目标：仅重构首页表单页文案与轻交互提示，不改后端能力与数据映射，实现“创始人越填越清楚自己为何需要 FA、为何需要 AI FA”的叙事闭环。

* 核心定位：MVP 只做一件事——帮助创业者 CEO 判断“这几家 VC 谁更适合你”，并将隐含价值主张“更低费率、更全数据、更透明动机”。

* 成功标准：首页每一步都清晰回答“为什么现在要填、填完得到什么、下一步有什么价值”，提交前形成强动机转化。

### 与目标的差距

* 现有首页语言偏“信息录入助手”，没有持续强化“为什么你现在需要 FA/AI FA”。

* 步骤提示更偏填写说明，缺少“机会成本/时间窗口/信息差”这类用户决策压力与行动动机。

* 当前 CTA 偏功能描述（如“下一步/进入结果页”），未形成“继续填写=降低融资沟通风险”的心理收益。

## Proposed Changes

### 文件 1：`/Users/maha/vctea-nextjs/app/page.tsx`

#### 变更目标

* 在不改变字段结构和提交映射的前提下，重写首页所有关键文案。

* 引入“AI FA 价值叙事”与“轻压迫式紧迫感”（不使用恐吓、羞辱表达）。

#### 具体改动（What / Why / How）

* 顶部品牌与主视觉文案重写

  * **What**：重写顶部 tag、H1、副标题。

  * **Why**：第一屏直接传达“你不是在填表，而是在降低融资沟通成本并提升匹配概率”。

  * **How**：将主叙事统一为“AI FA 帮你让投资人更快看懂你，帮你筛出更适配 VC”。

* 步骤文案体系重构（`steps` 常量）

  * **What**：重写四步的 `title/subtitle/helper`。

  * **Why**：每一步都要回答“这一段信息将如何影响机构判断”和“不填/乱填的代价”。

  * **How**：将 helper 改为“决策影响语句 + 填写建议”，让用户感到每一步都在减少信息不对称。

* 字段级标签、占位符与解释文案升级

  * **What**：重写关键字段 placeholder 与解释块（融资金额、runway 等）。

  * **Why**：减少用户“我不知道怎么填”的卡点，并持续强化 AI FA 的分析价值。

  * **How**：使用“对投资人可读”的输入模板与示例，加入“填得越清晰，匹配越准确”的反馈锚点。

* 错误提示与按钮文案转化导向

  * **What**：重写必填报错、步骤按钮与提交按钮文案。

  * **Why**：避免机械提示，改为“继续填写即可获得下一层判断价值”的推进语言。

  * **How**：将 CTA 表达为“继续完善画像/进入匹配判断”等更具结果感的措辞。

* 轻交互文案强化（不新增后端）

  * **What**：在现有结构中增加微文案触发点（如步骤说明块、进度语义）。

  * **Why**：实现“越填越想填”的连续反馈，不引入新接口风险。

  * **How**：复用当前 UI 组件，仅替换文案和少量状态文案，不新增字段、不改 payload。

## Assumptions & Decisions

* 已锁定 PRD 基准：以用户在对话中提供的截图内容为主，不以仓库历史方案覆盖其方向。

* 改动范围：仅首页表单页（`app/page.tsx`）；结果页、API、workflow 输入键保持不变。

* 风格决策：采用“价值压力 + 机会成本”表达，而非攻击性恐惧文案；目标是提升行动紧迫感与继续填写意愿。

* MVP 目标不变：只服务“VC 适配判断”这一核心功能，不扩展新功能点。

  * 提交按钮具备明确结果承诺感（不是通用“提交/下一步”）。

# VC查 用户视角极简 UX 重构计划

## Summary
- 目标：把当前“信息过载、解释过多”的体验改成“用户一眼能懂、愿意开始填写”的极简流程。
- 覆盖范围：输入页 `app/page.tsx` + 结果页 `app/result/page.tsx` 同步简化。
- 核心方向：轻量分步、去复杂进度、减少文案噪音、默认隐藏历史记录、结果页逐张卡片展示。

## Current State Analysis
- 输入页当前问题（基于 `app/page.tsx`）：
  - 顶部信息区过重（标签 + 大段说明 + 多个价值点），首屏认知负担高。
  - 完成度百分比 + 步骤宫格 + 多说明并存，用户注意力被“流程解释”分散。
  - 自动填写历史记录在主路径可见，打断首次填写路径。
- 结果页当前问题（基于 `app/result/page.tsx`）：
  - 既有总览/详情/调试并存，信息密度偏高。
  - 尚未按你给的 schema（`org_name`, `match_conclusion`, `confidence`, `match_highlights`, `attention_items`, `suggested_info`）做明确映射与排序优先级。

## Assumptions & Decisions
- 已确认决策：
  - 输入页采用“轻量分步”，不要复杂进度百分比与重解释。
  - 顶部只保留“标题 + 一句说明”。
  - 历史记录默认隐藏在次级入口。
  - 结果页一起改，改为逐张卡片。
  - 不保留“原始输出（调试）”区域。
  - `match_conclusion` 决定卡片排序与样式，排序优先级固定为：
    1) 推荐优先沟通
    2) 建议探索性接触
    3) 建议补充更多信息后再评估
    4) 当前匹配度有限，建议关注其他机构
    5) 暂不建议优先推进
- 约束保持：
  - 输入页提交到 workflow 的既有键名保持不变（如 `sai_dao`、`rong_zilun_ci` 等）。

## Proposed Changes

### 1) 输入页：`app/page.tsx`
- **结构简化**
  - 顶部仅保留：品牌短标签、主标题、一句说明；删除当前三枚价值标签块。
  - 移除“完成度 x/x + 百分比 + 进度条”信息。
  - 将步骤导航改为轻量样式：只显示“当前步骤标题 + 简短副标题”，不展示复杂步骤宫格。
- **交互简化**
  - 保留“上一步/下一步/提交”，但去掉所有非必要解释块，仅保留关键术语解释（融资金额、runway）。
  - 分步校验保持，仅在点击下一步或提交时给最短错误提示。
- **历史记录降干扰**
  - 将历史记录改为默认隐藏入口按钮（如“使用历史记录”），点击后展开列表。
  - 首屏不直接展示历史记录内容，避免打断首次用户决策。
- **视觉简化**
  - 继续维持“低边框 + 圆角 + 留白”，进一步减少装饰块数量。
  - 保留必要微动效（切换、hover、focus），但降低动效数量与视觉干扰。

### 2) 结果页：`app/result/page.tsx`
- **schema 映射落地**
  - 优先按你给的字段解析：
    - `org_name` -> 机构名
    - `match_conclusion` -> 结论（用于排序与样式）
    - `confidence` -> 置信度展示
    - `match_highlights` -> 匹配亮点
    - `attention_items` -> 关注事项
    - `suggested_info` -> 建议补充信息
  - 兼容当前旧字段解析作为兜底（不影响你这版 schema 直接可用）。
- **排序与样式策略**
  - 依据 `match_conclusion` 的优先级排序（高优先级卡片排前）。
  - 按结论档位设置卡片视觉状态（强调色/弱化色），让用户一眼区分优先级。
- **展示方式**
  - 改为纯“逐张卡片”信息流：每张卡片展示结论、置信度、亮点、关注事项、建议补充。
  - 去掉“原始输出（调试）”区。
  - 保留“返回修改表单 / 重新评估”两个核心操作。

### 3) 一致性
- 输入页和结果页统一简洁风格：
  - 信息密度下降
  - 文案更短
  - 主按钮层级清晰
  - 无技术化模块暴露给最终用户

## Verification
- 代码正确性：
  - `GetDiagnostics` 检查 `app/page.tsx` 与 `app/result/page.tsx` 无新增错误。
  - `npx eslint app/page.tsx app/result/page.tsx` 通过。
- 交互验证（本地预览）：
  - 输入页首屏“标题 + 一句说明”即可进入填写，不再出现复杂进度信息。
  - 历史记录默认隐藏，点击入口后可展开并正常回填。
  - 结果页按 `match_conclusion` 优先级正确排序，卡片样式与排序一致。
  - 结果页不再显示调试输出。
- 回归验证：
  - 从输入页提交后可正常跳转结果页并展示机构卡片。
